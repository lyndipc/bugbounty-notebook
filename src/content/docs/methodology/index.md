title: "Methodology"
---

Bug bounty hunting follows a repeatable process. These notes capture the workflow, program rules, and submission strategies that lead to accepted reports.

## High-Level Flow

1. **Scope Review** — Read the program policy, understand what's in/out of scope
2. **Reconnaissance** — Map the attack surface (subdomains, ports, endpoints, APIs)
3. **Vulnerability Discovery** — Test each surface area for known bug classes
4. **Exploitation** — Prove impact, chain bugs where possible
5. **Reporting** — Write a clear, reproducible report with evidence
6. **Follow-Up** — Respond to triager questions, provide clarification
